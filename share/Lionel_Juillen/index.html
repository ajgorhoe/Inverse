A:
It is already possible to plug in own optimisatin algorithms, but unfortunately the instructions for using the relevant library have not been written yet. Inverse is designed as an open code where all basic functionality can be accessed through appropriate library functions and any kind of functionality can be added, too. Because of insufficient development resources, manuals have been written only for using the program as-is, and not yet for using the Inverse library. In my current projects there is not much space for writing these manuals, so I can not forecast when they will actually be written. When there exists a strong enough interest on the side of some of our partners, the priority list can also be accommodated and specific things can be done earlier. Within our group I currently teach people how to link other code with Inverse by myself.
Parametrization ability is limited on what is offered by programme interfaces and on processing abilities which enable definition of more complex functionals on basis of data obtained from the direct analysis (e.g. using the expression evaluator or matrix and vector utilities). More elaborated interface is currently available only for Elfen (a FEM simulation program developed by Rockfield Software). For interfacing other programs the general file interface can be used.
Shape parametrisation utilities are not yet included, except some splines manipulation functions, which can be useful for 2D shape optimisation (instructions for using  this are not contained in the latest web manuals, but you can download the PostScript version by clicking on this link).
I am currently developing a 2D and 3D shape parametrisation approach, which would be quite general and also compatible with sensitivity analysis requirements. I expect that 2D shpe parametrisatin module will be built in the shell by the end of this year. 3D modul should be available in a year and a half. You can find some basic ideas of the approach in the following article (click here to download it in the pdf format - 200 kB):

  •  13. I. Gresovnik, T. Rodic and D. Jelovsek (2001). Simple Two Stage Transforms Designed for Optimisation of Shape in Forming Processes, in A.M. Habraken (ed.): Proceedings of the 4th International ESAFORM Conference on Material Forming held in Liege, Belgium, vol. 2, pp. 831-834. University of Liege.


    Installation and running instructions:
 

  1. Unpack the directory contained in the zip file which you have downloaded by clicking on the link. The unpacked directory  is named ighome. You can put this directory anywhere you want or rename it, but do not change its structure or delete files contained in it.
  2. After unpacking the directory, set the environment variable IGHOME to hold the directory path (e.g. if you have unpacked the directory to "C:\Program Files\ighome", then this must be the value of the variable IGHOME).  On MS Windows you can usually set the environment variables by invoking the Control Panel (Windows menu Start/Settings/Control Panel) and double-clicking the System icon. In the window which appears, you choose the Environment box. In the Variable entry at the bottom you input IGHOME and in the Value entry the appropriate directory name (note that if the directory name contains spaces, it must be in double quotes).
  3. If you want to run the program from any directory, you can add the bin subdirectory of the unpacked directory to the Path variable. You can do this by choosing the Path variable in the System variables menu in the same Environment box where you have set the IGHOME variable. You can also copy the program inverse.exe from the bin directory to any directory which is already contained in the Path variable or run the program every time by using the complete path of the executable. The location of executable doesn't matter, but for proper function there must exist a program resource directory with appropriate structure and its name must be the value of the environment variable IGHOME.
You can run the program e.g. by typing
inverse com_filename
where com_filename is the name of the command file in which you give the program instructins what to do. Syntax and available commands for the command file are described in Inverse manuals and in other documentation , e.g. the Quick Introduction to Inverse. You can first try to run some training examples.

    You will get some help by invoking the program by

inverse -h
This will not be of much use, however, because the help system is not yet elaborated. If you don't get help in this way, you should set a valid path to a html browser (e.g. Netscape) in the file $IGHOME\scripts\browse.bat and a path to a text editor (e.g. Notepad) in the file $IGHOME\scripts\edit.bat
